Approach

A calm, repeatable path from messy requirements to systems that stay observable, correctable, and production-ready.

Discover

02

03

Develop

Why this direction

The live site already has strong content and a disciplined "spec sheet" system. The problem is not craft. It is tone: mono labels, hairline drafting furniture, and agent schematics as the hero read as terminal-first, not human-first.

Thesis: Warm Authority

Feel like a trusted senior architect in the room: calm, specific, human. Professional without performing "AI aesthetic."

  • References: Stripe Press, boutique architecture studios, Linear marketing (warmer)
  • Keep: evidence density, case study depth, outcomes
  • Drop: omnipresent mono labels, index numbers as identity, schematic-as-hero

Foundation changes

  • Type: Fraunces (display) + Plus Jakarta Sans (UI/body)
  • Color: warm ink / cream paper + copper accent (not cold graphite + ember redline)
  • Shape: softer 18px radius, pill CTAs, subtle grain
  • Depth: soft multi-layer shadows + warm surface steps
  • Hero: photo + idea-led headline, not name + circuit diagram

Still AI architect

  • Proof stats and production language stay
  • Featured work still shows system diagrams, softer and secondary
  • Approach section keeps Discover / Design / Develop as narrative cards
  • No neon, purple gradients, particles, or "robot" chrome
